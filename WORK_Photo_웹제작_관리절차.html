파일 수정 시 주의사항

⚠️ index.html 저장 시 반드시 확인!
파일형식: 모든 파일 (*.*) 선택
인코딩: UTF-8 선택
파일명: index.html (index.html.txt 가 되지 않도록 주의!)
🚨 절대 하지 말 것!
- 기존 파일을 백업 없이 덮어쓰기
- GitHub에서 직접 편집 후 로컬 파일과 동기화 안 하기
- APK 빌드 후 파일명 없이 저장

파일 버전 관리 방법

1
수정 전 백업

현재 index.html → backup/index_20260610.html 로 복사

2
Claude AI로 코드 수정

Claude에서 수정된 코드 → "코드 복사" → 메모장에 붙여넣기

3
로컬 테스트

index.html 파일을 브라우저로 열어서 동작 확인

4
GitHub 업로드

GitHub 저장소 → 기존 파일 삭제 → 새 파일 업로드 → Commit

5
배포 확인

3~5분 후 https://faraoh77777.github.io/shaheen-photo/ 접속 확인

유지보수 체크리스트

수정 작업 전 체크리스트

  • 현재 index.html 백업 폴더에 날짜로 복사했는가?
  • Claude AI 대화에서 현재 버전 아티팩트 확인했는가?
  • 수정할 기능 요구사항 명확히 정리했는가?
  • 로컬에서 브라우저 테스트 가능한 환경인가?

수정 작업 후 체크리스트

  • PC 브라우저에서 기본 동작 확인했는가?
  • 모바일 Chrome에서 사진 촬영 테스트했는가?
  • 사진 저장(다운로드) 정상 동작 확인했는가?
  • GitHub에 새 파일 업로드 완료했는가?
  • GitHub Pages URL에서 반영 확인했는가?
  • 모바일 앱(WebView)에서도 정상 동작하는가?

APK 빌드 체크리스트

  • App.js의 WebView URL이 최신 GitHub Pages URL인가?
  • Expo 계정 로그인 상태인가? (eas login)
  • EAS_NO_VCS=1 환경변수 설정했는가?
  • 이전 APK를 apk-releases 폴더에 보관했는가?
  • 새 APK를 모바일에서 테스트했는가?

자주 발생하는 문제 빠른 해결

증상해결
GitHub Pages 업데이트 안됨기존 파일 삭제 후 새 파일 업로드
파일명이 .html.txt로 저장됨저장 시 파일형식 "모든 파일" 선택
eas 명령어 없음npm install -g eas-cli
git 오류로 빌드 실패set EAS_NO_VCS=1 후 재실행
모바일에서 공유 안됨Chrome 또는 Safari로 직접 접속
앱이 기본 Expo 화면App.js WebView 코드 확인 후 재빌드
